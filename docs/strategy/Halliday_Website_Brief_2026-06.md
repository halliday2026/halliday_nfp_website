## What this brief is — and isn't

**It IS:** the sitemap, page-by-page section structure, content direction, placement of already-locked language, global elements, and build-critical reminders.

**It is NOT:**
- **Visual/design direction** (look, typography, color, layout feel) — that's the separate downstream creative-direction pass. Don't conflate messaging-unhappy with design-unhappy: the messaging is locked; this brief arranges it. The *look* is a different conversation.

## Operating rules that carry into every page (from Thread 1)

1. Lead with the customer + outcome. Never lead with "custom build" or "AI."
2. Custom build is the HOW — discoverable in the body (the Data Work page), never the flag.
3. Deliver with AI, never sell the AI.
4. Never market "cheaper." Watch the three quiet doors: a **price tag near the hero**, **"easy/simple" as the loudest note**, **speed-as-the-pitch**.
5. Website value prop = a real person + peace of mind (done-for-you), not hosting or price.
6. Hero flies the wedge now, tightens onto data at Gate 1. Emphasis phases; identity doesn't.

---

## The single most important structural requirement

**Hero leads with websites; the body and the data pages must establish the data moat from launch.**

Otherwise "lead with websites now" silently becomes "be the website guy forever." Every page below is structured to honor this — the home page asserts the data case in its body, and the Data Work page carries the expertise hard.

## Page-by-page

### Home
**Goal:** in the first screen, make the right ED feel seen; show the front door (websites); keep data visible and credible; route to the calculator (soft) and the discovery call (hard).

1. **Hero.** Locked hero line + funnel-in-one-line subhead. Primary CTA "Book a discovery call," secondary CTA "See what scattered data is costing you" → calculator. *Keep modular for the Gate-1 re-weight.*
2. **Who this is for (self-identify).** Short "does this sound like you?" list using the messy-middle signals: data in 3+ places, a recent grant win, no in-house data person, board decks assembled by hand. This is the fix for Amber's "reads as for large orgs" — speaks to the ED directly.
3. **Before / After.** Condensed version of the data-tier emotional core (full version lives on Data Work). This is where the data moat starts asserting on the home page.
4. **The four pillars.** As locked: one web-entry pillar + three data-outcome pillars (raise more / report faster / prove impact).
5. **Calculator teaser.** Strong teaser → dedicated calculator page (keeps home focused; gives the calculator its own shareable URL).
6. **Proof teaser.** EHL case study snippet (with permission) → Proof page. Social proof of the *data* work specifically, reinforcing the moat.
7. **Final CTA.** Discovery call.

### Data Work — THE MOAT PAGE
**Goal:** establish Halliday as a serious data consultancy, not a web shop. Make the $7,500 engagement obviously worth it. This page does the heaviest lifting against "be the website guy forever."

1. **Header.** The data-outcome promise (raise more / report faster / prove impact).
2. **Full Before / After.** The complete emotional-core framing.
3. **Three data pillars, expanded.** Each with the real-life pain and the payoff.
4. **The productized offer.** The fixed-scope donor analysis (provisional name *Donor Insight Engagement* — see cleanup note). What's included (retention, lapsing/at-risk, lifecycle segmentation, top-donor trends — *with insights and recommended actions, not a data dump*), and the outcome. **Anchor at the $7,500 floor** ("engagements start at $7,500") to qualify out tire-kickers and value-anchor.
5. **What working together looks like.** Brief: discovery call → engagement. Sets the free-discovery-conversation as the entry.
6. **The custom-build differentiator.** This is where "custom build" is *allowed* to live — in the body. Name the real moat: the transformation logic and the "what should we measure?" expertise, **not the dashboard UI.**
7. **CTA.** Discovery call.

### Managed Websites — the wedge
**Goal:** the fast-yes. Done-for-you + peace-of-mind register. **Never cheap-led.**

1. **Header.** A professional web presence, handled for you. Peace-of-mind promise.
2. **What you get.** Built and kept running by a real person who makes sure it's always current and working — one less thing to think about.
3. **Who it's for.** Nonprofits *and* the orbit (consultants, coaches, grant writers). This is where breadth lives — in the body, never the hero.

6. **CTA.** Get started / contact.

## One judgment call for you (pricing visibility)

A standalone public pricing page isn't obviously right, so I'm flagging rather than deciding. My recommendation:

- **Website pricing: transparent** on the Managed Websites page ($75/mo). It's a fast-yes; transparency reduces friction.
- **Data work: show the floor** ("engagements start at $7,500") on the Data Work page — enough to qualify and value-anchor — with the discovery call as the CTA. No public price grid for the consulting.
- **No separate pricing page** unless you want one. Reason: a dedicated pricing page tends to pull the $75/mo to prominence and re-anchor the brand cheap (operating rule #4). Pricing lives *on the relevant service page*, in context.

If you'd rather keep the data floor off the site entirely (discovery-call-only), that's defensible too — it trades qualification for a softer first impression. Your call before final copy.
